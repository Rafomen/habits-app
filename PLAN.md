# План: Habits App — Дисциплина дня

## Суть приложения
Веб-приложение для жесткого соблюдения распорядка дня на честном слове.
Ты сам задаёшь задачи, сам назначаешь цену выполнения и штраф за срыв.
Цель — заработать виртуальный 1,000,000$. В конце дня отчитываешься
о времени на развлечения и своём настроении. Графики растут вместе.

---

## Пользовательский путь (User Flow)

### Шаг 1. Регистрация / Вход
- Email + пароль
- Попадаем на пустой экран: "У тебя нет плана дня" + кнопка "Создать план дня"

### Шаг 2. Онбординг (первый вход)
1. **Во сколько ты просыпаешься?** (например, 07:00)
2. **Во сколько ты ложишься спать?** (например, 23:00)
3. Приложение рисует временную шкалу от пробуждения до сна
   (например, 07:00 — 23:00 = 16 часов)

### Шаг 3. Заполнение плана дня
- Пользователь заполняет отрезки времени задачами
- Форма добавления задачи:
  - **Название** — свободный текст
    (например: "Принять холодный душ после пробуждения",
    "Не курить весь день", "Сделать зарядку")
  - **Привязка ко времени** — один из двух вариантов:
    - **Временная рамка** — задача привязана к конкретному слоту
      (например, 07:00–07:30)
    - **В течение дня** — задача без привязки ко времени, должна быть
      выполнена до конца дня (например, "Не курить весь день")
  - **Награда за выполнение** — $ (например, 1000$)
  - **Штраф за невыполнение** — $ (например, 3000$)
- Если временной слот уже занят другой задачей — приложение скажет
  "Это время уже занято, выбери другое"

### Шаг 4. После заполнения дня
- Приложение предлагает: **"Сделать этот план на каждый будний день?"**
  - Да — план копируется на Пн–Пт
  - Нет — план только на сегодня
- Затем спрашивает: **"Какой план на Сб–Вс?"**
  - Можно создать отдельный план на выходные
  - Или пропустить (без плана на выходные)

### Шаг 5. Постановка цели
- Приложение спрашивает: **"Какая у тебя цель?"**
- Пользователь вводит (например, "Заработать 1,000,000$")
- Приложение считает: при заработке ~X$ в день (сумма наград за все задачи)
  тебе потребуется Y дней без ошибок для достижения цели
- Вывод: "Для выполнения этого гола надо 100 дней безошибочного
  выполнения ежедневно"

### Шаг 6. Ежедневный цикл

#### Утро — отчёт за вчера
При входе в приложение первым делом — отчёт за предыдущий день:

**Вопрос 1: Сколько минут ты просидел в играх / соцсетях вчера?**
- Пользователь вводит число
- Приложение сравнивает с разрешённым временем (60 мин по умолчанию)
- Если превысил → **двойной штраф**:
  - За каждую лишнюю минуту списывается штраф с баланса
  - (конкретную формулу штрафа уточним, но суть: нарушил — плати вдвойне)

**Вопрос 2: Твоё настроение перед сном по 10-балльной шкале?**
- Пользователь выбирает от 1 до 10
- Сохраняется для графика

После ответов → показывается итог вчерашнего дня:
- Сколько задач выполнено / не выполнено
- Заработано $ / потеряно $
- Текущий баланс
- Разрешённое время на развлечения сегодня

#### День — выполнение задач
- Список задач на сегодня с галочками
- Задачи со временем — отсортированы по дедлайну
- Задачи "в течение дня" — в отдельном блоке снизу
- Просроченные (время вышло, галочки нет) — подсвечены красным
- Баланс и прогресс до цели — видны на экране

#### Вечер — подготовка ко сну
- Автоматическое завершение дня в указанное время сна
- (или手动 кнопка "Завершить день")

## Механики в деталях

### Баланс ($)
- Начальный баланс: 0$
- Каждая выполненная задача: +награда$ (задаётся пользователем)
- Каждая невыполненная задача: −штраф$ (задаётся пользователем)
- Превышение времени на развлечения: двойной штраф
- Баланс может уходить в минус
- Прогресс-бар: X% пути до 1,000,000$

### Наградное время (минуты на игры/соцсети)
- Базовое разрешённое время: 60 минут в день
- Невыполненные задачи уменьшают разрешённое время
  (конкретную формулу привязки уточним)
- Если превысил разрешённое время → двойной штраф с баланса на следующий день
- Долг переносится: если "недодал" время, остаток переходит на завтра

### Графики (вкладка отчётов)
- **График настроения** — по оси Y: 1–10, по оси X: дни
- **График успешности** — % выполнения задач по дням
- Оба графика рядом — чтобы видеть корреляцию
  (чем лучше выполняешь план, тем лучше настроение)

## Структура БД

### users
- id, email, password_hash, name
- balance (float), carried_debt (float)
- wake_time, sleep_time
- goal_amount (например, 1000000)
- created_at

### tasks (задачи)
- id, user_id, title
- time_start (nullable — null = задача на весь день)
- time_end (nullable)
- reward_amount (float) — награда за выполнение
- penalty_amount (float) — штраф за невыполнение
- applies_weekdays (bool), applies_weekends (bool)
- is_active (bool) — активна ли задача в текущем плане
- created_at

### daily_task_completions (отметки выполнения)
- id, task_id, user_id, date
- is_completed (bool), completed_at

### daily_reports (отчёты за день)
- id, user_id, date
- tasks_total, tasks_completed
- earned (float), lost (float)
- games_social_minutes (int) — сколько реально просидел
- allowed_minutes (int) — сколько было разрешено
- games_penalty (float) — штраф за превышение
- mood (int, 1-10)
- created_at

### balance_log (история баланса)
- id, user_id, date, amount, type (reward/penalty/games_penalty), description

## Экраны / Страницы

### 1. Auth (Login / Register)
- Вход и регистрация

### 2. Onboarding (только при первом входе)
- Шаг 1: время пробуждения и сна
- Шаг 2: создание первого плана дня
- Шаг 3: предложение будни/выходные
- Шаг 4: постановка цели

### 3. Morning Review (первый экран каждого нового дня)
- "Отчёт за вчера"
- Ввод: минуты в играх/соцсетях
- Ввод: настроение (1–10)
- Итог: заработано / потеряно / баланс

### 4. Dashboard (основной экран дня)
- Текущий баланс и прогресс до цели (% и $)
- Разрешённое время на развлечения сегодня
- Список задач на сегодня (с галочками)
- Задачи по времени сверху, задачи "на день" снизу
- Просроченные — красным

### 5. Plan Editor
- Редактирование задач (добавить / удалить / изменить)
- Визуальная шкала времени (таймлайн дня)
- Контроль пересечений временных слотов

### 6. Reports (отчёты)
- График настроения (1–10 по дням)
- График успешности (% выполнения по дням)
- История: что выполнил / что нет / за что списано
- Детализация по каждому дню

### 7. Settings
- Время пробуждения / сна
- Цель ($)
- Разрешённое время на развлечения (базовое)
